# 《真言地牢》游戏设计分析完善计划

## 目标
对《真言地牢》GDD/PRD v1.0 进行全面分析、完善，输出完整的从开发到上线的阶段规划和策略报告。

## 阶段分解

### Stage 1 — 深度研究（并行）
- **研究A**: 教育游戏（Edutainment/严肃游戏）市场分析、竞品分析（如《Duolingo》游戏化机制、《Hades》Roguelike设计、《Splelling Bee》等词汇游戏）、商业模式
- **研究B**: 技术可行性验证 — Whisper.cpp离线语音识别在不同平台的实际表现、8000词知识图谱构建工作量、Unity 2D URP跨平台性能、SQLite移动端性能
- **研究C**: 游戏设计风险评估 — 核心机制复杂度分析、玩家学习曲线、输入方式接受度、留存率挑战

### Stage 2 — 分析报告撰写
- 基于Stage 1研究成果，撰写完整的分析完善报告
- 包含：设计亮点、风险与问题、完善建议、完整开发阶段规划、团队配置、预算估算、上线策略、商业化方案
- 输出为 Markdown 格式

### Stage 3 — 文档格式化输出
- 加载 `docx` skill
- 将最终报告转换为专业排版的 .docx 文件
- 交付给用户

## 技能加载
- Stage 1: `deep-research-swarm`
- Stage 2: `report-writing`
- Stage 3: `docx`

## 输出路径
- 分析文件：`/mnt/agents/output/真言地牢_设计分析与开发策略.md`
- 最终文档：`/mnt/agents/output/真言地牢_设计分析与开发策略.docx`
